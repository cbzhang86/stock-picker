# A股智能选股系统

**13 个直连数据源 · 30+ 量化因子 · 短线/长线双策略 · 回测引擎 · 自学习权重**

全市场5205只实时扫描，多因子加权评分 → 尾盘推荐Top N（含仓位分配）。  
兼容 Claude Code / OpenClaw / Hermes 等 AI 编程助手。

## 🚀 快速开始

```bash
# 1. 安装
pip install mootdx requests pandas numpy scikit-learn cachetools akshare pyyaml

# 2. 短线策略（每日14:50运行，~4分钟出结果）
python scripts/eod_stock_picker.py --mode short

# 3. 查看模型状态和历史表现
python scripts/eod_stock_picker.py --status

# 4. 回测验证
python scripts/run_backtest.py --mode short --start 2025-06-01 --end 2026-06-11
```

## 🔧 数据源

| 数据 | 来源 | 协议 | 稳定性 |
|------|------|------|--------|
| K线+财务 | **mootdx**（通达信） | TCP 7709 | ⭐ 永不封IP |
| 实时行情 | **腾讯财经** | HTTP | ⭐ 不封IP |
| 同花顺热点 | **10jqka** | HTTP | ⭐ 零鉴权73ms |
| 北向资金 | **hexin.cn** | HTTP | ⭐ 零鉴权 |
| 板块归属 | **东财push2** | HTTP限流 | ✅ 已限流+会话复用 |
| 龙虎榜 | **东财datacenter** | HTTP限流 | ✅ 已限流 |
| 资金流 | **akshare**（大单） | HTTP | 仅当日数据 |
| 代码列表 | **akshare** | HTTP | 本地缓存加速 |

### 数据源失效保护

系统内置降权机制：当某个数据源不可用时，对应因子自动中性化为50分，其权重重分配给其他活跃因子。评分不压缩，不会因单源失效而整体失效。

## 📈 短线策略评分流程

```
全市场5205只
  ↓ 初筛（非ST / 成交额>3000万 / 非涨跌停）
预过滤 ~4900只  
  ↓ 5维预评分（流动性30 + 活跃度20 + 动量15 + 风险20 + PE15）
Top 200 进入详评
  ↓ 资金流 + K线 + 技术评分 + RPS横排
评分排序
  ↓ 防凑数（与第1名分差>20分裁掉）
仓位分配（评分加权，单只上限40%/下限10%）
  ↓
最终推荐 1-3 只（含板块归属+龙虎榜信息）
```

### 权重（8因子）

| 因子 | 权重 | 说明 |
|------|:---:|------|
| 主力资金流 | **27%** | 大单交易汇总净流向 |
| 动量/RPS | **15%** | 全市场横截面百分位 |
| 技术形态 | **15%** | 6维系统化评分 |
| 量价配合 | **10%** | 量比 |
| 风险过滤 | **10%** | ST/成交额/涨跌停 |
| 北向资金 | **10%** | 当日中观方向 |
| 同花顺热点 | **8%** | 题材归因标签 |
| 龙虎榜 | **5%** | 上榜+机构净买入 |

## ⚙️ 回测说明

- K 线全部来自 **mootdx 真实历史数据**
- 以 **次日开盘价成交**，计入滑点（0.1%）和佣金（万三）
- T+1/T+5 收益、胜率、夏普、最大回撤均基于真实价格
- 输出权益曲线块状图、月度收益表、因子归因表
- **已知限制**：资金流/北向不可回溯（当日大单数据）；仅覆盖最近约600个交易日

## 💡 常见问题

**Q: 适合什么场景？**  
A: 尾盘选股（14:50-15:00），T+1 开盘卖出。单次运行约 4-5 分钟。

**Q: 需要国内网络？**  
A: mootdx（TCP）和腾讯API不受地域限制，国内国外都通。东财部分接口在国外可能受限，系统会自动降权。

**Q: 能用于实盘交易吗？**  
A: 当前版本仅为决策辅助工具。建议先通过回测和模拟盘验证策略有效性。

**Q: 如何修改因子权重？**  
A: 编辑 `config.yml` 的 `short_term.weights` 或 `long_term.weights`。权重总和需 = 1.0。
